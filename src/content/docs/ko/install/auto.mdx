Astro를 설치할 준비가 되었나요? 이 가이드를 따라 `create astro` CLI 도구를 사용해보세요.

:::note[수동으로 Astro를 설치하려면?]
[단계별 수동 설치 가이드](/ko/install/manual/)를 읽어주세요.
:::
#### 기본 요구 사항

- **Node.js** - `v16.12.0` 또는 상위 버전.
- **텍스트 에디터** - [VS Code](https://code.visualstudio.com/)와 [공식 Astro 확장 프로그램](https://marketplace.visualstudio.com/items?itemName=astro-build.astro-vscode)을 함께 사용하는 것을 추천해요.
- **터미널** - Astro를 제어할 명령줄 인터페이스(CLI)가 필요합니다.

#### 설치하기

`create astro`는 Astro 프로젝트를 시작하는 가장 빠른 방법입니다. 새 프로젝트를 시작하기 위한 모든 단계를 안내하고 필요한 경우 공식 템플릿을 고를 수 있도록 도와줍니다. 또한, [`--template` 인자를 사용하면 GitHub에 있는 기존 프로젝트를 활용할 수 있습니다.](#시작-템플릿)

:::tip[웹에서 미리보기]
Astro 프로젝트를 쉽고 간편하게 브라우저 안에서 시험해 볼 수 있습니다. [astro.new](https://astro.new/)를 방문하여 시작 템플릿 중 하나를 고르고 새로운 프로젝트를 시작해보세요.
:::

## 1. 설치 마법사 실행하기

다음 명령어를 터미널 안에서 실행하면 설치 마법사의 도움을 받을 수 있습니다:

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  # npm으로 새 프로젝트 만들기
  npm create astro@latest
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  # pnpm으로 새 프로젝트 만들기
  pnpm create astro@latest
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  # yarn으로 새 프로젝트 만들기
  yarn create astro
  ```
  </Fragment>
</PackageManagerTabs>

`create astro` 는 어디서나 실행할 수 있는 명령어입니다. 만약 디렉터리가 비어 있지 않다면, 설치 마법사가 새 프로젝트를 위한 디렉터리를 자동으로 만들어줍니다.

모든 게 정상적으로 동작했다면, "Liftoff confirmed. Explore your project!" 문구와 함께 다음 단계를 안내받습니다. `cd` 명령어를 입력해 새 프로젝트 디렉터리로 이동하면 새로운 Astro 프로젝트를 사용할 준비를 끝마치게 됩니다.

설치 마법사의 도움을 받는 도중에 `npm install` 단계를 건너뛰었다면 다음으로 넘어가기 전에 직접 패키지를 설치하는 과정을 잊지 마세요!

## 2. Astro 시작하기 ✨

Astro에는 프로젝트 개발에 필요한 모든 것을 갖춘 개발 서버가 내장되어 있습니다. `astro dev` 명령어를 입력하면 로컬 개발 서버와 함께 새로운 웹사이트가 실행됩니다.

모든 시작 템플릿은 `astro dev` 명령어를 실행하는 스크립트가 함께 제공됩니다. 원하는 패키지 관리자를 사용해 명령어를 실행하고 Astro 개발 서버를 시작해보세요.

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  npm run dev
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  pnpm run dev
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  yarn run dev
  ```
  </Fragment>
</PackageManagerTabs>


서버가 정상적으로 실행됐다면 [http://localhost:3000/](http://localhost:3000/)에서 동작하는 프로젝트를 확인할 수 있습니다.

Astro는 `src/` 디렉터리 안의 변경 사항을 자동으로 감지합니다. 따라서 개발 중에 변경 사항을 적용하기 위해 서버를 재시작할 필요가 없습니다.

브라우저에서 프로젝트를 열 수 없는 경우에는 `dev` 명령을 실행한 터미널로 돌아가 오류가 발생했는지 확인해보세요. 프로젝트가 위에서 제공한 URL과는 다른 주소에서 실행되고 있을 가능성도 있습니다.

## 시작 템플릿

템플릿을 사용해 더욱 간편하게 Astro 프로젝트를 시작할 수 있습니다. `create astro` 명령 실행 시 `--template` 인자를 건네주고 [공식 예제](https://github.com/withastro/astro/tree/main/examples) 또는 다른 GitHub 저장소의 `main` 브랜치를 템플릿으로 활용해보세요.

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  # 공식 예제를 템플릿으로 사용하기
  npm create astro@latest -- --template <example-name>

  # GitHub 저장소의 main 브랜치를 템플릿으로 사용하기
  npm create astro@latest -- --template <github-username>/<github-repo>
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  # 공식 예제를 템플릿으로 사용하기
  pnpm create astro@latest --template <example-name>
  
  # GitHub 저장소의 main 브랜치를 템플릿으로 사용하기
  pnpm create astro@latest --template <github-username>/<github-repo>
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  # 공식 예제를 템플릿으로 사용하기
  yarn create astro --template <example-name>
  
  # GitHub 저장소의 main 브랜치를 템플릿으로 사용하기
  yarn create astro --template <github-username>/<github-repo>
  ```
  </Fragment>
</PackageManagerTabs>

`main` 브랜치가 아닌 다른 브랜치를 템플릿으로 사용하려면 `--template` 인수를 다음과 같이 적어야 합니다: `<github-username>/<github-repo>#<branch>`.

이 밖에도 블로그, 포트폴리오, 개발 문서, 랜딩 페이지 등을 위한 다양한 종류의 템플릿이 [테마 쇼케이스 페이지](https://astro.build/themes/)를 통해 제공되며, 부족하다면 GitHub 검색을 통해 새로운 템플릿을 찾아볼 수도 있습니다.
