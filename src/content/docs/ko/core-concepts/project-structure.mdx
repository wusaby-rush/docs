`create astro`로 생성한 새로운 Astro 프로젝트에는 몇 개의 파일과 폴더가 포함되어 있습니다. 나머지는 직접 생성하여 Astro의 기존 파일 구조에 추가할 수 있습니다.

여기서는 Astro 프로젝트가 어떻게 구성되어 있는지, 그리고 새로운 프로젝트에 포함되는 몇 가지 파일에 관해 설명합니다.

## 디렉터리 및 파일

Astro는 프로젝트에 고유한 디렉터리 구성을 사용합니다. 모든 Astro 프로젝트의 최상위에는 다음과 같은 디렉터리와 파일이 포함되어야 합니다.

- `src/*` - 당신의 프로젝트 소스 코드 (컴포넌트, 페이지, 스타일 등)
- `public/*` - 소스 코드가 아니거나 처리되지 않은 에셋 파일들 (폰트, 아이콘 등)
- `package.json` - 프로젝트 매니페스트
- `astro.config.mjs` - Astro의 설정 파일 (권장)
- `tsconfig.json` - TypeScript 설정 파일 (권장)

### 디렉터리 구조 예시

일반적인 Astro 프로젝트의 디렉터리는 다음과 같은 형태입니다.

<FileTree>
- public/
  - robots.txt
  - favicon.svg
  - social-image.png
- src/
  - components/
    - Header.astro
    - Button.jsx
  - layouts/
    - PostLayout.astro
  - pages/
    - posts/
      - post1.md
      - post2.md
      - post3.md
    - index.astro
  - styles/
    - global.css
- astro.config.mjs
- package.json
- tsconfig.json
</FileTree>

### `src/`

프로젝트 대부분의 소스코드는 `src/`디렉터리에 저장됩니다. 여기에는 아래 내용들이 포함됩니다.

- [Pages](/ko/core-concepts/astro-pages/)
- [Layouts](/ko/core-concepts/layouts/)
- [Astro 컴포넌트](/ko/core-concepts/astro-components/)
- [Framework Components](/ko/core-concepts/framework-components/)
- [Styles & CSS](/ko/guides/styling/)
- [Markdown & MDX](/ko/guides/markdown-content/)

Astro는 `src/`에 있는 파일을 처리하고, 최적화하고, 번들링하여 브라우저에 표시되는 웹 사이트를 생성합니다. 정적인 `public/` 디렉터리와 달리 `src/`에 있는 파일은 Astro에 의해 빌드되고 처리됩니다.

일부 파일(예: Astro 컴포넌트)은 그대로 브라우저로 전송되지 않고 정적인 HTML로 변환됩니다. 다른 파일(예: CSS)은 브라우저로 전송되지만, 성능을 위해 최적화되거나 다른 CSS 파일과 함께 번들로 제공될 수 있습니다.

### `src/assets`

[`src/assets/`](/ko/guides/assets/) 디렉터리는 Astro에서 처리하는 에셋(이미지 등)을 저장하는 데 사용할 권장 폴더입니다. 이 폴더는 필수는 아니며 특별한 예약 폴더도 아닙니다.

### `src/components`

컴포넌트는 HTML 페이지에서 재사용할 수 있는 코드 단위로, Astro 컴포넌트나 React, Vue와 같은 UI 프레임워크 컴포넌트가 이에 해당합니다. 프로젝트의 모든 컴포넌트를 이 폴더에 모아두는 것이 일반적입니다.

이것은 Astro 프로젝트에서 흔히 사용되는 컨벤션이지만, 꼭 지켜야 하는 것은 아닙니다. 원하는 대로 컴포넌트를 구성하세요!

### `src/content`

`src/content/` 디렉터리는 [content collections](/ko/guides/content-collections/)를 저장하기 위해 예약되어 있는 선택 사항인 컬렉션 구성 파일입니다. 이 폴더 안에는 [content collections](/ko/guides/content-collections/)외의 다른 파일은 허용되지 않습니다.

### `src/layouts`

[Layouts](/ko/core-concepts/layouts/)은 일부 콘텐츠를 더 큰 페이지 레이아웃으로 감싸는 특별한 종류의 컴포넌트입니다. [Pages](/ko/core-concepts/astro-pages/)나 [Markdown & MDX](/ko/guides/markdown-content/)으로 작성된 페이지의 레이아웃을 정의하는 데 자주 사용됩니다.

`src/components`와 마찬가지로 이 디렉터리는 일반적인 관례이지만 꼭 필요한 것은 아닙니다.

### `src/pages`

[Pages](/ko/core-concepts/astro-pages/)는 사이트에 새로운 페이지를 만드는 데 사용되는 특별한 종류의 컴포넌트입니다. Astro 컴포넌트, 사이트 컨텐츠를 나타내는 마크다운 등이 페이지가 될 수 있습니다.

:::caution
`src/pages`는 Astro 프로젝트에서 **필수적인** 하위 디렉터리입니다. 이 디렉터리가 없으면 사이트에는 페이지나 경로가 없습니다!
:::

### `src/styles`

CSS 또는 Sass 파일을 `src/styles` 디렉터리에 저장하는 것이 흔히 사용되는 컨벤션이지만 반드시 그럴 필요는 없습니다. 스타일이 `src/` 디렉터리의 어딘가에 있고 올바르게 가져온 것이라면 Astro는 스타일을 처리하고 최적화합니다.

### `public/`

`public/` 디렉터리는 Astro의 빌드 프로세스 중에 처리할 필요가 없는 파일과 에셋을 위한 디렉터리입니다. 이 파일들은 그대로 빌드 폴더에 복사됩니다.

이러한 동작들로 인해 `public/` 디렉터리는 이미지 및 폰트와 같은 일반적인 에셋이나 `robots.txt` 및 `manifest.webmanifest`와 같은 특수한 파일들을 저장하기에 최적의 장소가 됩니다.

CSS 및 JavaScript를 `public/` 디렉터리에 넣을 수 있지만 최종 빌드에서 번들로 제공되지 않고 최적화되지 않는다는 점에 유의해야 합니다.

:::tip
일반적으로 작성한 CSS나 자바스크립트는 `src/` 디렉터리에 저장해야 합니다.
:::

### `package.json`

이 파일은 JavaScript 패키지 관리자가 종속성을 관리하는 데 사용되는 파일입니다. 또한 Astro를 실행하는 데 일반적으로 사용되는 스크립트를 정의합니다(예: `npm start`, `npm run build`).

`package.json`에는 `dependencies`와 `devDependencies`라는 [두 종류의 종속성](https://docs.npmjs.com/specifying-dependencies-and-devdependencies-in-a-package-json-file)을 지정할 수 있습니다. 대부분의 경우, Astro는 빌드 시 모든 종속성을 요구하고 패키지 관리자는 둘 다 설치하기 때문에 둘 다 동일하게 작동합니다. 우선 `dependencies`에 모든 의존성을 포함시키고, 특별히 필요한 경우에만 `devDependencies`를 사용하는 것을 권장합니다.

프로젝트에 대한 새 package.json 파일을 만드는 데 도움이 필요하다면 [수동 설정 지침](/ko/install/manual/)을 참조하세요.

### `astro.config.mjs`

이 파일은 모든 스타터 템플릿에서 생성되며, Astro 프로젝트의 설정 옵션을 포함하고 있습니다. 여기에서 사용할 통합, 빌드 옵션, 서버 옵션 등을 지정할 수 있습니다.

설정에 대한 자세한 내용은 [Configuration Reference](/ko/reference/configuration-reference/#article)를 참조하세요.

### `tsconfig.json`

이 파일은 모든 스타터 템플릿에서 생성되며, TypeScript의 설정 옵션을 포함하고 있습니다. 일부 기능(예: npm 패키지 가져오기)은 `tsconfig.json` 파일이 없으면 에디터에서 완전히 지원되지 않습니다.

설정에 대한 자세한 내용은 [TypeScript](/ko/guides/typescript/) 가이드를 참조하세요.
